# Story: Configuração da Estrutura Monorepo e Roteamento Base

**ID:** `feature/1.1-monorepo-foundation`
**Sprint:** 1 (Fundação)
**Prioridade:** 🔴 CRÍTICA
**Status:** ✅ COMPLETE

## 1. Descrição
Como desenvolvedor, preciso de uma estrutura de pastas organizada e um ponto de entrada único (`index.php`) para que o sistema possa gerenciar rotas de API e servir o frontend de forma consistente, seguindo o padrão Front Controller.

## 2. Contexto Técnico
- **Arquitetura:** Monorepo MVSC.
- **Ambiente:** PHP 8.1 Vanilla.
- **Roteamento:** O `index.php` na raiz deve atuar como dispatcher.

## 3. Critérios de Aceite
1.  **Estrutura de Pastas:** Criar diretórios `/backend`, `/frontend`, `/tests`, `/backend/Controllers`, `/backend/Services`, `/backend/Models`, `/backend/Core`.
2.  **Front Controller:** `index.php` deve capturar requisições via `$_SERVER['REQUEST_URI']`.
3.  **API Health Check:** Implementar roteamento para `GET /api/health` retornando `{ "status": "ok", "timestamp": "..." }`.
4.  **CORS & Headers:** O `index.php` deve configurar headers base para respostas JSON e permitir CORS básico.
5.  **Configuração de Servidor:** Criar um arquivo `.htaccess` (Apache) ou instrução para servidor embutido do PHP que redirecione requisições não-arquivos para o `index.php`.

## 4. Definição de Pronto (DoD)
- [x] Estrutura de pastas reflete o `ARCHITECTURE.md`.
- [x] Endpoint `/api/health` retorna 200 OK com JSON válido.
- [x] Código segue as diretrizes de Clean Code e SOLID.
- [x] Teste básico de roteamento implementado em `/tests`.

## 5. Notas de Implementação
- Manter o KISS (Keep It Simple, Stupid). Não utilizar bibliotecas de roteamento complexas; um simples `switch` ou mapeamento de array no `index.php` é suficiente para o início.
- O diretório `/frontend` deve conter um `index.html` temporário ("Hello Atacadão") para validar o serviço de arquivos estáticos.
