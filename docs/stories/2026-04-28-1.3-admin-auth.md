# Story: Autenticação Administrativa via Sessões Nativas PHP

**ID:** `feature/1.3-admin-auth`
**Sprint:** 1 (Fundação)
**Prioridade:** 🟡 ALTA
**Status:** ✅ COMPLETE

## 1. Descrição
Como administrador do sistema, preciso me autenticar para acessar a área de upload do catálogo, garantindo que apenas usuários autorizados possam modificar os dados de preço e faturamento.

## 2. Contexto Técnico
- **Arquitetura:** MVSC (Controller -> Service -> Model).
- **Segurança:** Uso de `session_start()` e `password_verify()` nativos do PHP 8.1.
- **Banco de Dados:** Tabela `admin_users` (id, username, password).
- **Conexão:** Utilizar `App\Core\Database::getInstance()`.

## 3. Critérios de Aceite
1.  **AuthController:** Criar `backend/Controllers/AuthController.php` com o método `login(request)`.
2.  **AuthService:** Criar `backend/Services/AuthService.php` para encapsular a lógica de verificação de credenciais.
3.  **Persistência:** Após login bem-sucedido, armazenar o `user_id` e `username` na global `$_SESSION`.
4.  **Middleware de Proteção:** Implementar método `AuthService::isAuthenticated()` para validar se a sessão está ativa.
5.  **Endpoints:** 
    - `POST /api/login`: Recebe JSON `{username, password}`.
    - `POST /api/logout`: Destrói a sessão.
    - `GET /api/auth/status`: Retorna se o usuário está logado.
6.  **Responses:** Retornar status HTTP apropriados (200 OK, 401 Unauthorized, 500 Error).

## 4. Definição de Pronto (DoD)
- [x] Login com credenciais válidas (`admin` / `admin123`) funciona e cria sessão.
- [x] Login com credenciais inválidas retorna 401.
- [x] Logout limpa a sessão corretamente.
- [x] Testes unitários para `AuthService` implementados em `/tests`.
- [x] Commits atômicos realizados via `caveman-commit`.

## 5. Notas de Implementação
- O `index.php` (Front Controller) deve ser atualizado para incluir a rota de login/logout.
- Garantir que `session_start()` seja chamado no início do `index.php`.
- Nomes de variáveis e métodos em Inglês (EN-US).
