# Story: Motor de Importação de Catálogo via CSV Legado

**ID:** `feature/1.4-catalog-import`
**Sprint:** 1 (Fundação)
**Prioridade:** 🔴 CRÍTICA
**Status:** ✅ COMPLETE

## 1. Descrição
Como administrador, quero fazer upload do relatório `.csv` gerado pelo Job interno para atualizar instantaneamente o catálogo de produtos e preços disponível para os operadores.

## 2. Contexto Técnico
- **Segurança:** Requer autenticação de sessão (`AuthService::isAuthenticated()`).
- **Parsing:** `fgetcsv` com delimitador `;`.
- **Banco de Dados:** Tabela `catalog`.
- **Sanitização:** Conversão de valores financeiros (PT-BR para SQL) e remoção de espaços.

## 3. Critérios de Aceite
1.  **Endpoint de Upload:** `POST /api/catalog/upload` protegido por autenticação.
2.  **Validação de Arquivo:** Validar MIME type `text/csv` e extensão.
3.  **Mapeamento de Colunas:** Extrair apenas `MERC`, `DIGITO`, `DESCRICAO`, `EMBALAGEM`, `ESTOQ EMB1`, `ESTOQ EMB9`, `PRECO VENDA`.
4.  **Processamento Atômico:** Limpar tabela existente e inserir novos dados em uma única transação/operação segura.
5.  **Sanitização:** 
    - Trim em todos os campos.
    - `PRECO VENDA`: Substituir `,` por `.` para garantir tipo `DECIMAL`.
6.  **Response:** Retornar JSON com `{ "success": true, "count": X }` onde X é o número de itens importados.

## 4. Definição de Pronto (DoD)
- [x] Upload de CSV válido substitui o catálogo no MariaDB.
- [x] Tentativa de upload sem login retorna 401.
- [x] CSV com colunas faltando retorna erro 400 amigável.
- [x] Testes unitários para `CsvImportService` garantem a sanitização correta.
- [x] Commits atômicos realizados via `caveman-commit`.

## 5. Notas de Implementação
- O arquivo CSV contém muitas colunas (30+), o parser deve localizar as colunas desejadas pelo nome no cabeçalho.
- Utilizar `fgetcsv($handle, 0, ';')`.
