# Story: Módulo de Operação (Frontend / Pedidos Assíncronos)

**ID:** `feature/1.5-operator-module`
**Sprint:** 1 (Fundação)
**Prioridade:** 🔴 CRÍTICA
**Status:** ✅ COMPLETE

## 1. Descrição
Como operador do sistema, preciso de uma interface intuitiva para buscar produtos no catálogo e montar pedidos de faturamento de forma rápida e assíncrona, garantindo que meus dados não interfiram nos pedidos de outros operadores.

## 2. Contexto Técnico
- **Interface:** `/frontend/operador/index.html`.
- **Tecnologias:** Bootstrap 5, jQuery, Font Montserrat.
- **Gestão de Estado:** `sessionStorage` para isolamento de abas.
- **API Backend:** Necessário `GET /api/catalog` para busca por MERC ou Descrição.

## 3. Critérios de Aceite
1.  **Interface do Operador:** Criar layout com cabeçalho institucional (conforme DESIGN.md) e tabela de itens selecionados.
2.  **Busca de Produtos:** Implementar input de busca que consulte o catálogo via backend.
3.  **Gestão de Itens:**
    - Adicionar item selecionado à lista.
    - Editar quantidade.
    - Remover item.
4.  **Cálculos Automáticos:** 
    - `R$ Total` por linha (Preço * Qtd).
    - `TOTAL GERAL` do pedido no rodapé da tabela.
5.  **Persistência Local:** Garantir que o `sessionStorage` mantenha os dados após Refresh da página.
6.  **Isolamento:** Abrir uma nova aba deve iniciar um pedido em branco (comportamento nativo do `sessionStorage`).

## 4. Definição de Pronto (DoD)
- [ ] Operador consegue pesquisar e adicionar produtos.
- [ ] Totais são calculados corretamente.
- [ ] Dados persistem no refresh mas são limpos ao fechar a aba.
- [ ] Código frontend limpo e comentado (EN-US).
- [ ] Commits atômicos via `caveman-commit`.

## 5. Notas de Implementação
- Implementar endpoint `GET /api/catalog?search=...` no `CatalogController`.
- Utilizar a fonte Montserrat para legibilidade operacional.
