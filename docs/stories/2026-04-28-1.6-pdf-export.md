# Story: Exportação de PDF e Geração de Código de Barras

**ID:** `feature/1.6-pdf-export`
**Sprint:** 1 (Fundação)
**Prioridade:** 🟡 ALTA
**Status:** ✅ COMPLETE

## 1. Descrição
Como operador, preciso gerar um documento PDF da cotação que seja idêntico ao modelo da planilha original, incluindo códigos de barras legíveis por scanners de mão, para seguir com o faturamento físico.

## 2. Contexto Técnico
- **Biblioteca de Exportação:** `html2canvas` + `jsPDF` (ou alternativas modernas como `jspdf-autotable` se aplicável).
- **Fonte de Barcode:** `Libre Barcode 39`.
- **Regra de Barcode:** `*MERC + DIGITO*` (asteriscos são necessários para o padrão Code 39).
- **Layout:** Seguir o cabeçalho institucional e as colunas definidas no PRD inicial.

## 3. Critérios de Aceite
1.  **Template de Impressão:** Criar um elemento HTML (pode ser oculto na tela) formatado exatamente como a planilha original (Logotipo Atacadão, Cliente, Data, Horário, Tabela de Itens).
2.  **Geração de Barcode:** A coluna de código de barras deve renderizar a fonte `Libre Barcode 39` com o valor concatenado corretamente.
3.  **Exportação Client-side:** Ao clicar em "Gerar Faturamento", capturar o template e gerar o download do PDF sem passar pelo servidor.
4.  **Fidelidade Visual:** O PDF deve manter as proporções e fontes (Montserrat e Barcode).
5.  **Data/Hora:** O PDF deve registrar o momento exato da geração.

## 4. Definição de Pronto (DoD)
- [ ] Botão "Gerar Faturamento" funcional.
- [ ] PDF baixado contém todos os itens da cotação.
- [ ] Códigos de barras aparecem corretamente com a fonte específica.
- [ ] Layout segue o padrão Atacadão.
- [ ] Commits atômicos via `caveman-commit`.

## 5. Notas de Implementação
- Incluir `html2canvas` e `jsPDF` via CDN.
- Utilizar `@font-face` para garantir que a fonte de código de barras seja incorporada na captura.
- Garantir que valores como "R$ Total" e "Total Geral" estejam destacados.
