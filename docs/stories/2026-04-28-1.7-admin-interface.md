# Story: Interface Administrativa (Dashboard de Upload)

**ID:** `feature/1.7-admin-interface`
**Sprint:** 1 (Fundação)
**Prioridade:** 🟡 ALTA
**Status:** 🏗️ IN PROGRESS

## 1. Descrição
Como administrador do sistema, preciso de uma interface protegida para realizar o login e fazer o upload do arquivo CSV de catálogo, garantindo que os dados de preço e produtos estejam sempre atualizados.

## 2. Contexto Técnico
- **Interface:** `/frontend/admin/index.html`.
- **Tecnologias:** Bootstrap 5, jQuery.
- **Segurança:** Integração com os endpoints `/api/login` e `/api/catalog/upload`.
- **UX:** Feedback de progresso e alertas de sucesso/erro via SweetAlert2.

## 3. Critérios de Aceite
1.  **Tela de Login:** Implementar formulário de login (username/password).
2.  **Dashboard de Upload:** 
    - Exibir área de upload após login bem-sucedido.
    - Input de arquivo restrito a `.csv`.
    - Botão de envio que dispara a importação.
3.  **Feedback Visual:** 
    - Exibir loading durante o processamento do CSV.
    - Mostrar resumo da importação (quantidade de itens processados).
4.  **Gestão de Sessão:** 
    - Verificar status de autenticação ao carregar a página.
    - Opção de Logout.
5.  **Persistência:** Redirecionar para login caso o usuário tente acessar o dashboard sem estar autenticado.

## 4. Definição de Pronto (DoD)
- [x] Login e Logout funcionais no frontend.
- [x] Upload de CSV válido atualiza o catálogo com feedback visual. (Integrado)
- [x] Interface segue o padrão visual Atacadão (DESIGN.md). (Dashboard completo com Sidebar/Navbar)
- [x] Código limpo e modularizado (EN-US).
- [x] Commits atômicos via `caveman-commit`.

## 5. Notas de Implementação
- Utilizar `FormData` no JS para enviar o arquivo via AJAX.
- Integrar SweetAlert2 para mensagens de erro/sucesso amigáveis.
